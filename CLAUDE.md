# Clinical Trial Search — Project Context for Claude Code

## What this project is

A production-grade AI system that lets patients search open-source clinical trials
by describing their symptoms in plain language. The core differentiator is a
QLoRA fine-tuned BioMistral-7B model that screens trial eligibility and ranks
results by confidence — replacing expensive GPT-4o calls with a domain-specific
open-source model deployed on Modal serverless GPU.

Built for public portfolio use. Goal: demonstrate full ML lifecycle at production
scale for Senior AI Engineer job applications.

---

## Architecture overview

```
Patient symptom query
  → SciSpacy NER  (extract medical entities)
  → MedlinePlus   (normalize to canonical condition)
  → pgvector ANN  (top-20 semantically similar trials)
  → BioMistral-7B (QLoRA fine-tuned, Modal GPU endpoint)
       screens each trial → {eligible, confidence, reason}
  → FastAPI        (ranked response)
  → React frontend (trial cards + eligibility explainer)
```

Offline training loop (separate from inference path):
```
ClinicalTrials.gov criteria
  → GPT-4o teacher (synthetic patient profiles + eligibility labels)
  → JSONL dataset (~240 examples)
  → QLoRA fine-tune on Colab T4 (3 epochs, ~45 min)
  → LoRA adapter (~40MB) pushed to HuggingFace Hub
  → Modal loads adapter at cold start
```

## Tech stack

| Layer | Technology | Notes |
|---|---|---|
| Base model | BioMistral/BioMistral-7B | Medical domain pretrained |
| Fine-tuning | QLoRA (PEFT + TRL SFTTrainer) | 4-bit NF4, r=16, ~0.5% params |
| GPU training | Google Colab T4 / RunPod A10G | Free or $0.25/hr |
| Model serving | Modal serverless A10G | $0.000595/sec, scales to zero |
| Adapter registry | HuggingFace Hub | Public portfolio artifact |
| Embeddings | text-embedding-3-large | 1536-dim, OpenAI |
| Vector store | pgvector (PostgreSQL ext) | ivfflat index, cosine similarity |
| NER | SciSpacy en_core_sci_lg | UMLS concept extraction |
| API | FastAPI + uvicorn | Async, pydantic validation |
| Cache | Redis | 24hr TTL, query fingerprint key |
| Observability | LangSmith | Traces, latency, weekly evals |
| Frontend | React + Tailwind | Vercel deploy |
| Orchestration | Airflow / AWS Batch | Weekly trial sync |
| Data sources | ClinicalTrials.gov v2, NCI, MedlinePlus | All free, no auth |

## Data models

```python
@dataclass
class Trial:
    nct_id: str
    title: str
    status: str                    # RECRUITING | COMPLETED | etc.
    conditions: list[str]
    interventions: list[str]
    eligibility_summary: str       # raw inclusion/exclusion text
    sponsor: str
    phase: str
    start_date: str
    locations: list[str]
    source: str                    # ClinicalTrials.gov | NCI Trials
    url: str

@dataclass
class ScreeningResult:
    nct_id: str
    title: str
    eligible: bool
    confidence: float              # 0.0 – 1.0
    reason: str                    # one sentence
    key_criteria_met: list[str]
    key_criteria_failed: list[str]
    latency_ms: float
```

## Fine-tuning quick reference

```bash
# 1. Generate labeled training data (~$5 OpenAI cost)
python models/finetune_pipeline.py --stage data

# 2. QLoRA fine-tune on Colab T4 (free) or RunPod
python models/oss_finetune.py --train --model BioMistral/BioMistral-7B --epochs 3

# 3. Evaluate adapter vs base model
python models/oss_finetune.py --eval --adapter ./outputs/qlora-biomistral/adapter

# 4. Push adapter to HuggingFace Hub (public portfolio artifact)
python models/oss_finetune.py --push --hub-repo yourusername/clinical-trial-eligibility-screener

# 5. Deploy to Modal
modal deploy serving/modal_endpoint.py
```

## Coding conventions

- Python 3.11+, type hints everywhere, dataclasses over dicts for data models
- All API clients use a shared `requests.Session` with `DEFAULT_TIMEOUT=15`
- Logging via `logging.getLogger(__name__)` — not print statements
- Wrap all external API calls in try/except, log warnings and continue (never crash the pipeline)
- Every new module needs a docstring with: what it does, dependencies, usage example
- Keep inference and training code strictly separated — nothing in `serving/` should import from `models/oss_finetune.py`
